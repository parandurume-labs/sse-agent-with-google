# 🛠️ Antigravity 에이전트 개발자 규칙 (developer_rules.md)

이 문서는 Antigravity 환경에서 본 프로젝트의 코드를 빌드하고 고도화할 때 준수해야 하는 코딩 표준과 품질 기준을 정의합니다.

## 🐍 Python 코딩 지침
- **버전**: Python 3.11 이상 호환성을 준수합니다.
- **포맷팅**: PEP 8 표준을 준수하며 가독성 높은 변수명과 명확한 함수 독스트링(Docstring)을 포함합니다.
- **의존성 관리**: 외부 라이브러리는 `requirements.txt`에 명시하고, 최소한의 검증된 패키지만 사용합니다.
- **예외 처리**: API 네트워크 통신 에러 및 파일 입출력 시 예외 처리를 빈틈없이 수행하고 상세 로그를 남깁니다.

## ☁️ Google Workspace & Cloud Platform 가이드라인
- **안전한 인증**: API 키 및 OAuth 토큰은 소스 코드에 하드코딩하지 않으며, 항상 `.env` 환경 변수 또는 GCP Secret Manager를 통해 로드합니다.
- **Google Cloud Functions**: 무상태(Stateless) 아키텍처를 원칙으로 하며, 단일 함수의 실행 시간 및 메모리 제한을 최적화합니다.
- **Google Apps Script**: 가용한 실행 할당량(Quota) 제한을 준수하도록 설계하며, 스크립트 실행 트리거를 적정 주기로 설정합니다.

## 🧪 테스트 및 디버깅
- 모든 백엔드 코드는 작성 후 목(Mock) 데이터를 활용한 유닛 테스트 혹은 모의 실행 검증을 거칩니다.
- 실제 Google 캘린더나 드라이브에 쓰기 작업을 수행하기 전, 대상 범위를 명확히 검증하는 디버깅 모드를 제공해야 합니다.
